## How to Use This List

* Start with **Roadmaps** to pick a learning path.
* Pick **one** ML or DL course, then supplement with **labs** and **projects**.
* Use **books** for depth, and **official tutorials** when learning a framework.
* Practice with **Kaggle** or small dataset projects. Share on GitHub.

> Legend: `[B]` Beginner, `[I]` Intermediate, `[A]` Advanced · `(free-to-audit)` may require sign-in but no payment.

## Math & Foundations

* **Linear Algebra, Probability, Optimization**

  * [Mathematics for Machine Learning (Deisenroth et al.)](https://mml-book.github.io) — Friendly intro to the math you’ll actually use. `[B]`
  * [An Introduction to Statistical Learning (ISL, 2e)](https://www.statlearning.com) — Modern applied stats for ML with labs. `[B][I]`
  * [The Elements of Statistical Learning (ESL)](https://hastie.su.domains/ElemStatLearn/) — Classic theory reference. `[A]`
  * [Understanding Machine Learning (Shalev-Shwartz & Ben-David)](https://www.cs.huji.ac.il/~shais/UnderstandingMachineLearning/copy.html) — Theory foundations of ML. `[I][A]`

## Project Ideas & Capstones

* **Beginner:** Titanic survival prediction, MNIST/CIFAR image classifier, spam detection, movie review sentiment, housing price regression.
* **Intermediate:** Named-entity recognition, object detection on custom data, recommendation system, time-series forecasting, tabular AutoML baseline vs manual models.
* **Advanced:** RL agent for a simple game, domain adaptation for CV, retrieval-augmented generation (RAG) with evaluation, multi-label classification with imbalanced data, MLOps pipeline with CI/CD and monitoring.

## Contributing

We welcome PRs that add **high-quality, free** resources.

1. Add a new item under the correct section with: **Title** · **Link** · **1-line why it’s useful** · **Level tag**.
2. Prefer **official sources**. Avoid mirrors or paywalled content.
3. Clearly mark items that are *(free-to-audit)* or require sign-in.
4. Keep descriptions short, neutral, and specific.
5. Broken links? Please open an issue.

*(Optional for maintainers: use a link checker and markdown linter via GitHub Actions.)*
